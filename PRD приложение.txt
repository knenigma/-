# PRD: «Куда, сколько и с кем?»

**Подзаголовок:** Найди простой план на сегодня.

**Тип документа:** Product Requirements Document (MVP)
**Рынок:** Казахстан
**Формат продукта:** Веб-приложение (mobile-first)

---

## 1. Обзор продукта

Веб-приложение, которое помогает человеку решить, чем заняться сегодня — если ему скучно, он устал, застрял в рутине, не знает куда пойти или как провести свободное время.

Приложение генерирует реалистичные мини-планы на день, вечер или выходные на основе:

- города
- настроения
- уровня энергии
- бюджета
- свободного времени
- формата досуга

## 2. Позиционирование

Это **не просто список мест**. Продукт должен давать готовые сценарии, которые человек реально может выполнить прямо сегодня. Примеры сценариев:

- «Спокойный вечер одному»
- «Перезагрузка после работы за 2 часа»
- «Выходной без большого бюджета»
- «Хочу выйти из дома, но без сложных планов»
- «Что-то новое, но недорого»
- «План для интроверта»
- «Я устал, но не хочу сидеть дома»
- «Простая встреча с другом»
- «Ненапряжное свидание»

## 3. Технологический стек

| Слой | Технология |
|---|---|
| Фреймворк | Next.js (App Router) |
| Язык | TypeScript |
| Стили | Tailwind CSS |
| UI-компоненты | shadcn/ui |
| Данные | Локальные статические данные (без backend) |
| Хранение сохранённых планов | localStorage |
| Деплой | Vercel |

**Явно вне скоупа MVP:** backend, авторизация, база данных, платежи, внешние API, AI API.

## 4. Основной пользовательский сценарий

1. Пользователь заходит на главную страницу.
2. Видит понятное описание продукта и кнопку **«Подобрать план»**.
3. Заполняет анкету параметров:
   - **Город:** Астана, Алматы, Шымкент, Другой город
   - **Настроение:** устал, скучно, одиноко, хочу что-то новое, хочу расслабиться, хочу выйти из дома, хочу встретиться с кем-то, хочу побыть один
   - **Уровень энергии:** низкий, средний, высокий
   - **Свободное время:** 1 час, 2 часа, 3 часа, вечер, целый день
   - **Бюджет:** бесплатно, до 3 000 ₸, до 5 000 ₸, до 10 000 ₸, гибкий бюджет
   - **Формат:** один, с другом, свидание, компания
   - **Предпочтение:** в помещении, на улице, смешанный вариант
4. После выбора параметров приложение генерирует **3 карточки с планами**.
5. Каждая карточка плана содержит:
   - название плана
   - короткое эмоциональное описание
   - таймлайн из 3–5 шагов
   - примерный бюджет
   - теги: для какого настроения/энергии/формата подходит
   - объяснение «почему этот план подходит пользователю»
   - запасной вариант (если пользователь слишком устал или погода плохая)
   - кнопки: **«Сохранить»**, **«Сгенерировать похожий»**, **«Скопировать план»**
6. Отдельная страница **«Сохранённые планы»** — планы хранятся через localStorage.
7. Кнопка **«Удиви меня»** — генерирует планы со случайными параметрами.
8. Кнопка **«Начать заново»**.

## 5. Логика рекомендаций

Простая **rule-based** система (без AI API на первом этапе).

### 5.1 Seed-данные: типы активностей

- спокойная кофейня
- прогулка
- книжный магазин
- музей/выставка
- спортзал/бассейн
- кино
- настольные игры
- парк
- прогулка у набережной
- недорогая еда
- восстановление одному
- встреча с другом
- недорогой городской маршрут
- домашний восстановительный план
- прогулка с красивым видом
- тихое место для перезагрузки

### 5.2 Факторы, влияющие на рекомендации

Система рекомендаций должна учитывать:

- город
- настроение
- уровень энергии
- бюджет
- свободное время
- формат досуга
- предпочтение: помещение/улица/смешанный вариант

### 5.3 Важные ограничения контента

- Для MVP **не** нужно показывать точные мероприятия, расписание, часы работы заведений или утверждать, что какие-то места точно открыты.
- Используются общие категории мест и реалистичные городские ориентиры.
- Формулировки — как **предложения**, а не как проверенная информация в реальном времени.

### 5.4 Примеры ориентиров по городам

**Астана:**
- прогулка у набережной
- ботанический сад
- тихая кофейня
- кино
- книжный магазин
- район Khan Shatyr
- район EXPO
- спокойная прогулка в центре

**Алматы:**
- прогулка в парке
- кофейня с видом
- район Kok-Tobe
- район Достык/Абая
- книжный магазин
- кино
- ботанический сад
- спокойный маршрут по центру

## 6. Дизайн

- **Mobile-first.**
- Современный, чистый, спокойный, немного premium.
- Много воздуха, мягкие отступы, скруглённые карточки, хорошая типографика.
- Ощущение дружелюбного lifestyle-помощника, а не корпоративного dashboard.
- Тёплый минималистичный стиль.
- Человечные микротексты, например:
  - «Без давления. Выбери то, что ближе к твоему состоянию.»
  - «Вот три простых плана, которые реально можно сделать сегодня.»
  - «Слишком устал? Выбери мягкую версию.»
  - «Не нужно идеального настроения, достаточно маленького плана.»
- Продуманные empty states и loading states.
- Приложение должно выглядеть завершённым и пригодным к использованию, а не как черновой демо-проект.

- Планы не должны выглядеть однообразно.

## 10. Требования к качеству кода

- Чистая структура проекта.
- Корректное использование TypeScript.
- Архитектура без излишнего усложнения.
- Переиспользуемые компоненты.
- Бизнес-логика рекомендаций отделена от UI.
- Комментарии только там, где они реально помогают.
- Архитектура позволяет легко заменить rule-based recommendation engine на AI API в будущем.
- Адаптивная вёрстка.
- Доступные (accessible) кнопки, labels и семантический HTML.
- Минимум хардкода внутри компонентов — данные выносятся отдельно.
